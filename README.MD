【技術実績書】Flutterによるテキストエディタ開発とAI活用プロセスの定式化

1. 開発成果物の概要

    プロダクト: 空間記述型テキストエディタ（Flutter/Windows）

    現在のステータス: Windows環境にて動作。

    設計思想: 「文字がない場所にはデータが存在しない」という既存エディタの制約を排し、ノートのように自由な位置へ記述・描画が可能な「空間優先」のデータ構造を構築。

    主要コンポーネント: * CustomPainterを用いた低レイヤー描画エンジン。

        テキストの論理座標（Row/Col）と描画座標（Offset）を動的に同期する座標変換ロジック。

        日本語IME（未確定文字列）のレンダリングおよびカーソル追従制御の独自実装。

2. 実装における技術的解決事項

    座標管理（AnchorPoint）: 描画データをピクセル単位ではなく「行・列」を基点とした相対座標系で管理。テキストの増減に伴う図形の動的追従を、自前の計算ロジックで実現。

    図形描画とテキストの共存: AIが生成した図形描画ロジック（L型線、テーブル、破裂等）を、本来「相容れない」性質を持つテキストエディタの描画スタックへ統合。各オブジェクトのハンドル操作やリサイズ制御をCustomPainter上で完遂。

    デバッグ管理: 描画時の期待値（全角文字の幅等）と実測値のズレをユニットテストおよび実機検証で特定し、1ピクセル単位の精度で修正。

3. AI（LLM）を活用した開発工程の最適化（AIマネジメント）
AIの誤生成や迷走を「人間がテックリードとして制御」するプロセスを確立。

    「先行合意型」プロンプティング: 大規模修正の際、コード出力前に「修正方針（Logic Design）」をAIに出力させ、論理の妥当性を確認してから生成を開始。

    「原点回帰」によるデバッグ制御: AIが誤修正を繰り返してコードが複雑化した際、直ちに初期の安定版へロールバック。変数のダンプログを優先的に解析させることで、根本原因の特定を最短化。

    コンテキストの明示的制御: AIエージェントの省略（サボり）に対し、ファイル指定（@filename）や全コード再提示を戦略的に行い、推論精度を維持。
